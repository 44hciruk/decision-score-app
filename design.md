# 決断スコア - UI/UX設計書

## アプリ概要
汎用型意思決定スコアリングアプリ。ユーザーが比較対象と評価基準を自由に設定し、ドラッグ&ドロップで順位付けを行い、100点満点のスコアで意思決定を支援する。

## 画面一覧（全7画面 + 課金モーダル）

### 1. ホーム画面（タブ: ホーム）
- **レイアウト**: 上部にアプリタイトル「決断スコア」、FlatListでプロジェクト一覧
- **コンテンツ**: 各プロジェクトカード（タイトル、日付、勝者名、スコア表示）
- **操作**: カードタップ → 履歴詳細画面、左スワイプ → 削除
- **FAB**: 右下に「＋」ボタン → テーマ設定画面へ遷移
- **空状態**: イラスト + 「最初のプロジェクトを作成しましょう」メッセージ

### 2. テーマ設定画面（モーダル/スタック）
- **レイアウト**: テキスト入力フィールド + テンプレート選択グリッド
- **コンテンツ**: タイトル入力（例：「今日のランチ」）、テンプレートカード一覧
- **テンプレート**: 飲食店、就職先、住居、旅行先、商品比較
- **操作**: テンプレート選択 → 項目自動入力、「次へ」ボタン → 候補入力画面

### 3. 候補入力画面（スタック）
- **レイアウト**: 動的テキスト入力リスト + 追加ボタン
- **コンテンツ**: 候補名入力フィールド（A, B, C...）
- **制限表示**: 「3/3 候補（無料版上限）」のプログレスバー
- **操作**: 「＋追加」ボタン（無料版3個超で課金モーダル表示）、「次へ」→ 評価項目設定画面

### 4. 評価項目設定画面（スタック）
- **レイアウト**: 動的テキスト入力リスト + 追加ボタン
- **コンテンツ**: 評価項目名入力フィールド（価格、味、雰囲気...）
- **制限表示**: 「5/5 項目（無料版上限）」のプログレスバー
- **操作**: 「＋追加」ボタン（無料版5個超で課金モーダル表示）、「スコアリング開始」→ 並び替え画面

### 5. 並び替え画面（スタック）
- **レイアウト**: 上部に進捗バー（1/4項目）、中央にドラッグ&ドロップリスト
- **コンテンツ**: 現在の評価項目名を大きく表示、候補をドラッグハンドル付きリストで表示
- **ドラッグUI**: 左端にハンドルアイコン（≡）、ドラッグ中はカードが浮き上がるエフェクト
- **操作**: ドラッグ&ドロップで順位変更、「次の項目へ」ボタン → 次の項目 or 結果画面

### 6. 結果表示画面（スタック）
- **レイアウト**: 上部に「結果発表」タイトル、中央に円形スコア、下部にランキング
- **円形スコア**: SVGで描画、カウントアップアニメーション（0→最終スコア、1000ms）
- **ランキング**: 1位は王冠アイコン付き大きめ表示、2位以降はコンパクト
- **確信度メッセージ**: 点差に応じた自動メッセージ表示
- **操作**: 「保存してホームへ」ボタン、「もう一度」ボタン

### 7. 履歴詳細画面（スタック）
- **レイアウト**: 結果表示画面と同様だがアニメーションなし
- **コンテンツ**: 保存済みプロジェクトの結果を再表示
- **操作**: 戻るボタン → ホーム画面

### 課金モーダル（オーバーレイ）
- **レイアウト**: ボトムシート形式
- **コンテンツ**: プレミアム機能一覧、月額/買い切りプラン選択
- **操作**: プラン選択 → 課金処理（モック）、「後で」→ モーダル閉じる

## キーユーザーフロー

### メインフロー（新規プロジェクト作成）
1. ホーム画面で「＋」FABタップ
2. テーマ設定画面でタイトル入力（またはテンプレート選択）
3. 候補入力画面で比較対象を入力（2〜3個）
4. 評価項目設定画面で評価基準を入力（2〜5個）
5. 並び替え画面で各項目ごとにドラッグ&ドロップで順位付け
6. 結果表示画面でスコア確認 → 保存

### 履歴確認フロー
1. ホーム画面でプロジェクトカードタップ
2. 履歴詳細画面で過去の結果を確認

## ナビゲーション構造

```
(tabs)/
  index.tsx          → ホーム画面（プロジェクト一覧）
  settings.tsx       → 設定画面（課金状態、テーマ切替等）

(stack)/
  new-project.tsx    → テーマ設定画面
  candidates.tsx     → 候補入力画面
  criteria.tsx       → 評価項目設定画面
  ranking.tsx        → 並び替え画面（ドラッグ&ドロップ）
  result.tsx         → 結果表示画面
  history-detail.tsx → 履歴詳細画面
```

## データ永続化
- **AsyncStorage** を使用（ローカルのみ、サーバー不要）
- プロジェクトデータはJSON形式で保存
- キー: `@decision_score_projects`

## 課金機能（UIモック）
- 実際のStoreKit/Google Play Billing連携はストア公開時に実装
- アプリ内ではプレミアム状態のトグルで動作確認可能に
- 無料版の制限（候補3個、項目5個、保存3個）をUIで表現
